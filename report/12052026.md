### Verifikasi reachability dan koreksi
Test pertama dari Chrome remote ke http://100.70.169.121:8090 timeout. Ternyata mesin ini punya dua identitas Tailnet sekaligus, daemon Tailscale di WSL native (100.70.169.121, hostname desktop-br2dlt9 linux) plus daemon Tailscale di Windows host (100.83.246.82, hostname desktop-br2dlt9-1 windows) yang sudah jalan 5 hari. Mode networking mirrored WSL2 bikin paket WireGuard masuk ke stack Windows duluan, jadi cuma identitas Windows yang bisa terima trafik dari Tailnet, identitas WSL praktis ghost. Port binding Docker (0.0.0.0:8090 dalam WSL) juga cuma surface ke Windows sebagai 127.0.0.1:8090 lewat mirrored mode loopback, bukan ke adapter Tailscale wintun. Akibatnya netsh portproxy 0.0.0.0 plus firewall rule scoped Tailnet di Windows tetap timeout karena netstat di Windows tidak punya listener real di 100.83.246.82:8090.

Solusi yang berhasil, `tailscale serve --bg --http=PORT http://127.0.0.1:PORT` di Windows host untuk keempat port. Tailscaled jadi proxy native di interface Tailscale dan forward ke 127.0.0.1 yang mana di-handle mirrored mode ke Docker dalam WSL. Bypass total konflik portproxy plus mirrored binding. Test dari peer Tailnet lain (kebetulan WSL identitas 100.70.169.121 counts as separate peer) return HTTP 200 baik via IP maupun via MagicDNS.

URL yang valid untuk dibagi ke anggota tim di tailnet gsp.engineerteam, pakai identitas Windows desktop-br2dlt9-1, bukan desktop-br2dlt9.
1. Frontend Autocrawl http://desktop-br2dlt9-1:8090 atau http://100.83.246.82:8090

Netsh portproxy plus rule Autocrawl-Tailscale-Inbound yang sempat dicoba sudah ditarik karena tidak diperlukan, tailscale serve sudah cover semua. Verifikasi netsh interface portproxy show v4tov4 kosong. Setup-port-proxy.ps1 dipertahankan di repo sebagai fallback.

### Aksi yang diambil
Lima aksi.
1. Diagnose timeout Chrome remote di 100.70.169.121:8090, ketemu dua identitas Tailnet co-exist (WSL linux dan Windows host) dengan mirrored mode kasih trafik Tailnet ke Windows-side aja.
2. Coba bridging via netsh portproxy 0.0.0.0:8090 ke 127.0.0.1:8090 plus firewall rule scoped 100.64.0.0/10 Private, gagal karena mirrored binding hanya surface di loopback Windows bukan di wintun.
3. Setup tailscale serve di Windows host untuk 4 port (8090, 8081, 3030, 9090) yang forward ke 127.0.0.1:PORT, bypass mirrored mode quirk.
4. Verifikasi dari peer Tailnet (WSL identity 100.70.169.121 ke Windows identity 100.83.246.82) return HTTP 200 baik via IP maupun MagicDNS hostname.
5. Cleanup netsh portproxy plus firewall rule eksperimen yang tidak terpakai, update report dengan URL final.

### Eskalasi
Tidak ada. Opsi peningkatan, kalau butuh HTTPS plus port standar 443, tailscale serve --bg https / http://127.0.0.1:8090 bisa diaktifkan supaya frontend muncul di https://desktop-br2dlt9-1.tailba2cde.ts.net tanpa nomor port. Sertifikat di-issue otomatis lewat Let's Encrypt via Tailscale, butuh enable HTTPS di admin console tailnet dulu. Belum dilakukan karena akses HTTP via IP plus port sudah memenuhi kebutuhan remote saat ini.
